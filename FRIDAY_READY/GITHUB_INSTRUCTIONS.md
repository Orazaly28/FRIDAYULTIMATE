# 📋 ИНСТРУКЦИЯ: Как получить готовый APK через GitHub

## ⏱️ Время: 15 минут | Сложность: Лёгкая

---

## ШАГ 1: Скачайте проект (уже готов)

**Файл проекта:** [FRIDAY_MOBILE.zip](../outputs/FRIDAY_MOBILE.zip)

Скачайте и распакуйте на телефоне или ПК.

---

## ШАГ 2: Создайте GitHub аккаунт (если нет)

1. Откройте https://github.com/signup
2. Введите email, пароль, имя пользователя
3. Подтвердите email
4. **Готово!** Вы на главной странице GitHub

---

## ШАГ 3: Создайте новый репозиторий

### На телефоне:
1. GitHub → кнопка **"+"** (правый верхний угол) → **"New repository"**
2. **Repository name:** `FRIDAY-Assistant`
3. **Description:** `AI Voice Assistant`
4. Выберите **"Public"** (чтобы работали Actions)
5. ✅ Галочка: **"Add a README file"**
6. Нажмите **"Create repository"**

### На ПК:
То же самое через веб-интерфейс github.com

---

## ШАГ 4: Загрузите файлы проекта

### Способ A — Через веб-интерфейс (проще):

1. В вашем репозитории нажмите **"Add file"** → **"Upload files"**
2. Перетащите ВСЕ файлы из папки `FRIDAY_MOBILE` (или выберите через кнопку)
3. Внизу напишите: `Initial commit - FRIDAY project`
4. Нажмите **"Commit changes"**
5. Дождитесь загрузки (1-3 минуты)

### Способ B — Через GitHub Mobile App (быстрее на телефоне):

1. Установите **GitHub** из Google Play
2. Войдите в аккаунт
3. Откройте репозиторий `FRIDAY-Assistant`
4. Нажмите **"⋮"** → **"Upload files"**
5. Выберите все файлы из `FRIDAY_MOBILE`
6. Commit → Push

---

## ШАГ 5: Запустите автоматическую сборку

1. В репозитории перейдите во вкладку **"Actions"** (верхнее меню)
2. GitHub покажет предупреждение: **"Workflows aren't being run..."**
3. Нажмите зелёную кнопку **"I understand my workflows, go ahead and enable them"**
4. Выберите workflow **"Build F.R.I.D.A.Y. APK"** (слева)
5. Справа нажмите **"Run workflow"** → выберите **"Branch: main"** → **"Run workflow"**
6. Появится жёлтый кружок 🟡 — сборка началась!

---

## ШАГ 6: Дождитесь завершения (8-12 минут)

**Что происходит:**
- GitHub устанавливает Java, Android SDK
- Компилирует 15 Kotlin-модулей
- Собирает APK
- Загружает результат

**Статус:**
- 🟡 Жёлтый — в процессе
- ✅ Зелёный — успех! Переходите к Шагу 7
- ❌ Красный — ошибка (пришлите скриншот Боссу)

---

## ШАГ 7: Скачайте готовый APK

1. После завершения (зелёная галочка ✅) кликните на название запуска (например, "Build F.R.I.D.A.Y. APK #1")
2. Прокрутите вниз до раздела **"Artifacts"**
3. Увидите: **"FRIDAY-APK"** (~15-20 MB)
4. Нажмите на него — начнётся скачивание ZIP-архива
5. Распакуйте ZIP → внутри будет **`app-debug.apk`**

---

## ШАГ 8: Установите на телефон

1. Скопируйте `app-debug.apk` на телефон (если качали на ПК)
2. Откройте файл через файловый менеджер
3. Разрешите установку из неизвестных источников:
   - Settings → Security → **"Install unknown apps"**
   - Или: Settings → Apps → Chrome/Browser → **"Install unknown apps"** → включить
4. Нажмите **"Установить"**
5. Дождитесь завершения → **"Открыть"**

---

## ✅ ГОТОВО! Теперь у вас F.R.I.D.A.Y. на телефоне

### Первый запуск:
1. Предоставьте все разрешения (Микрофон, Телефон, Контакты, etc.)
2. Включите **Accessibility Service**:
   - Settings → Accessibility → **"F.R.I.D.A.Y."** → включить
3. Скажите: **"Привет, Пятница"** или **"Friday, activate Veronica protocol"**

---

## 🆘 Если что-то не работает

### Ошибка в Actions (красный крестик):
- Откройте лог ошибки (кликните на название задачи)
- Скриншот → отправьте Боссу
- Я исправлю за 5 минут

### Файлы не загружаются в GitHub:
- Проверьте размер файлов (каждый должен быть < 100 MB)
- Попробуйте загружать по частям
- Или используйте GitHub Desktop (ПК)

### APK не устанавливается:
- Проверьте версию Android (нужна 7.0+)
- Удалите старые версии приложения
- Очистите кэш установщика пакетов

---

## 💡 БОНУС: Повторная сборка

После любых изменений в коде:
1. Загрузите новые файлы в репозиторий
2. Actions запустятся автоматически
3. Или запустите вручную: Actions → Run workflow

**Каждая сборка займёт ~10 минут.**
